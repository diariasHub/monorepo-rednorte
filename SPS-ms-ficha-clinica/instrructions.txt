# Instrucciones de Gestión y Uso - MS-FICHA-CLINICA

Este documento proporciona una guía rápida sobre el despliegue con Docker, la gestión de la base de datos PostgreSQL en Neon y cómo interactuar con la API.

## 1. Despliegue con Docker

El microservicio está configurado para ejecutarse de forma aislada mediante Docker Compose.

### Comandos de Gestión
*   **Iniciar todo**: `docker compose up --build -d`
*   **Ver estado**: `docker compose ps`
*   **Ver logs**: `docker compose logs -f app`
*   **Detener**: `docker compose stop`
*   **Eliminar contenedores**: `docker compose down`

## 3. Guía de Interacción (CRUD Básico)

La API sigue los principios REST. A continuación, un ejemplo de flujo básico para **Observaciones**.

### Crear Observación (POST)
```bash
curl -X POST http://localhost:8001/api/v1/observations \
     -H "Content-Type: application/json" \
     -d '{
       "patientId": "P001",
       "encounterId": "E001",
       "code": "8480-6",
       "value": 120.0,
       "unit": "mmHg"
     }'
```

### Consultar Observación (GET)
*   Ver todas: `GET http://localhost:8001/api/v1/observations`
*   Ver por ID: `GET http://localhost:8001/api/v1/observations/{id}`

### Actualizar/Eliminar
Para más detalles sobre otros módulos (Historia Clínica, Notas, Condiciones, Procedimientos, Encuentros), consulta la carpeta `/docs`.
